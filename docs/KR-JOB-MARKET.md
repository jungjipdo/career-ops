## 3. 한국 채용 방식 특성

### 채용 경로 우선순위

```
지인 추천 > 대규모 공채 > 수시 채용
```

- **공채**: 대기업 위주. 초중반에 서류 접수하는 것이 유리
- **수시**: 스타트업/중견기업. 원티드, 점핏 등 플랫폼 활용
- **추천**: 커피챗, 커리어리(https://www.careerly.co.kr/community) 등 네트워킹
- **계단식 이직**: 쟁쟁한 스타트업 → 중견 → 대기업 으로 단계적 이직 권장
- **이상적 재직 기간**: 3~5년. 잦은 이직은 마이너스

### 신입에게 바라는 역량

1. **개발 호기심** — 기본 언어 1개의 스펙을 잘 이해하고 구사 가능한지
2. **성장 욕구** — 자기계발 의지가 확실한지
3. **전공 이해도** — 전공 지식을 꾸준히 쌓고 있는지

### 주의해야 할 채용 공고 (Red Flags)

- 개발환경, 업무 설명, 스킬셋이 자세히 기술되어 있지 않은 공고 → 업무 체계 미비 가능성
- 대형 IT 회사에서 지나치게 많은 언어/스택 요구 → 업무 범위 불명확
- 주니어에게 특정 서드파티 종속 프로덕트만 시키는 곳 → 성장 제한

## 4. 한국 이력서 작성 가이드

### 필수 항목

- 인적사항: 이름, 연락처, 이메일, 생년월일, 병역여부
- **지원분야를 명확히** 기재
- 총 개발 경력
- 근무 경력 (개발 관련)

### 작성 원칙

1. **최신 이력을 가장 위에** 배치
2. **경력에 기술 스택을 함께** 표기 (예: `예약 시스템 개발 및 MSA 전환 (Java, Spring/JPA, MQ)`)
3. **지원 공고의 자격요건 스킬셋**과 일치하는 부분 강조
4. **분야 공통점 강조** (커머스 경험은 없지만 PG 연동 경험이 있다 등)
5. **기술 숙련도 표기**: 장황하게 늘어놓기보다 핵심 스택에 집중
6. **학력**: 연도, 학교, 전공, 졸업여부 정도만

### 추가 어필 요소 (있으면 좋은 것)

- 오픈소스 기여 이력
- 해커톤 입상
- 교육이력 (신입에게 유효)
- 아티클 번역/발행
- **실제 서비스 런칭 + 사용자 지표** (데모/이미지보다 강력)
- 블로그/깃허브는 경쟁력이 있어야 함 (단순 복사나 AI 활용은 독이 될 수 있음)

### 탈락을 부르는 케이스

- 달라진 것 없이 재지원
- 기술 일관성/전문성/깊이 부족
- 눈에 띄는 이력 없음 (옆그레이드만)
- 실제 프로젝트 투입/공헌도 낮음
- 개발 vs 운영 밸런스 극단적

## 5. 포트폴리오 작성 가이드

### 핵심 원칙

1. **최신 이력순**으로 정리
2. **고객에게 설명하듯** 쉽게 핵심만 기술 (product 링크 포함)
3. **본인의 역할** 자세히 기술하되 부풀리지 않기
4. **면접 질문을 유도**하는 방향으로 작성 (기술적 질문을 예측)
5. **트러블 슈팅 경험** 중심

### 트러블 슈팅 기록 프레임워크

```
어떤 문제를 맞닥뜨렸나?
→ 어떻게 해결하였는가?
→ 올바른 문제해결 과정을 고민했는가?
→ 접근 방식은 무엇이었고 합리적으로 유추하였는가?
→ 원인 → 과정 → 행동 → 결과 → 배운 점
```

### 포트폴리오 포함 내용

- 개발 목표와 의도 (무엇을 해결하려고 했고, 무엇을 제공했나)
- 주요 기능, 시스템 아키텍처 (시각화 권장: excalidraw 등)
- 본인이 담당한 영역과 기여도
- GitHub repo
- 형식: PDF 또는 Google Docs (Notion은 비추천). 서비스는 외부 접근 가능한 상태로.

### 깃허브 관리

- **깃 잔디**: 꾸준한 열정의 증거
- **프로필 꾸미기 참고**:
  - https://github.com/sproogen/modern-resume-theme
  - https://github.com/rahuldkjain/github-profile-readme-generator

## 6. 자기소개서

### 포함해야 할 내용

> "나는 무엇을 중요하게 바라보는 개발자" — 자신을 표현하는 한 줄

1. **지원동기** — 왜 이 회사/포지션인지
2. **관심 분야** — 무엇을 공부하고 있는지
3. **문제해결능력** — 팀 단위 성과 중심, 어떤 문제를 인식하여 해결했는지
4. **기여 포인트** — 나의 역량이 회사에 어떻게 기여할 수 있는지

### 커리어 관리 포인트

- 오픈소스 활동
- 세미나 연사 경험
- 책 집필, 팀 프로젝트
- 서비스 운영 중 문제해결과 성과
- **실패를 통해 배운 점** 정리
