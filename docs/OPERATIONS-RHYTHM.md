# Career-Ops 운영 리듬 가이드

> 단순히 공고를 분석하는 도구가 아니라, **지속 가능한 구직 운영 시스템**으로 활용하기 위한 리듬.

---

## 주간 운영 리듬

| 주기 | 시간 | 활동 | 명령어 |
|------|------|------|--------|
| **매일** | 5분 | 새 공고 스캔 | `/career-ops scan` |
| **수요일** | 30분 | Shortlist 리뷰 — Quick Screen으로 분류 | 트래커 확인 + 평가 |
| **금요일** | 1시간 | 지원 스프린트 — "즉시 지원" 공고 집중 | `/career-ops pdf` + 지원 |
| **월 1회** | 1시간 | Score Calibration — 패턴 분석, 필터 조정 | 결과 리뷰 |

---

## 일일 루틴 (5분)

```bash
# 1. 새 공고 스캔
/career-ops scan

# 2. 스캔 결과 확인
/career-ops tracker
```

- 새로운 공고가 pipeline에 추가됨
- **행동 없음** — 그냥 쌓아두기

---

## 수요일: Shortlist 리뷰 (30분)

```bash
# 1. 이번 주 쌓인 공고 확인
/career-ops tracker

# 2. 관심 공고에 Quick Screen 실행 (A+B블록만)
(채용공고 URL 붙여넣기)

# 3. Score에 따라 상태 분류
#    3.5+ → "즉시 지원" 또는 "네트워킹 먼저"
#    3.0-3.4 → "시장 관찰" 또는 "증빙 보강 후"
#    3.0 미만 → "SKIP"
```

**팁:** 한 번에 5-10개 Quick Screen 돌려서 빠르게 걸러내기. 전체 분석(Deep Dive)은 금요일에.

---

## 금요일: 지원 스프린트 (1시간)

```bash
# 1. "즉시 지원" 상태 공고 확인
/career-ops tracker

# 2. 각 공고에 Deep Dive (전체 A-F 분석)
(채용공고 URL 붙여넣기 → 전체 파이프라인)

# 3. PDF 생성
/career-ops pdf

# 4. Priority Lane (Score 4.0+ 인 경우)
/career-ops contacto     # LinkedIn 연락처 + 메시지
/career-ops deep          # 기업 심층 리서치
/career-ops interview-prep # 면접 준비 (면접 잡힌 경우)

# 5. 지원서 제출 (사용자가 직접!)
/career-ops apply         # 폼 작성 도우미
```

**팁:** 한 번의 스프린트에 2-3개 지원이면 충분. 양보다 질.

---

## 월간: Score Calibration (월 1회, 1시간)

### 체크리스트

- [ ] 이번 달 평가한 공고 수 확인
- [ ] 평균 Score 분포 분석
- [ ] "즉시 지원" → 면접 전환율 확인
- [ ] 반복적으로 나오는 갭(Gap) 패턴 분석
- [ ] portals.yml 스캔 쿼리 조정 (너무 넓으면 좁히기, 너무 좁으면 키워드 추가)
- [ ] title_filter 조정 (불필요한 공고 많으면 negative 키워드 추가)
- [ ] Archetype별 성과 확인 (어떤 유형이 잘 맞는지)

### 질문하기

```
"이번 달 applications.md를 분석해서:
1. 어떤 Archetype이 가장 점수가 높았는지
2. 반복되는 Gap이 뭔지
3. portals.yml 필터를 어떻게 조정하면 좋을지
알려줘."
```

## 사이클 요약

```
매일 스캔 (5분)
    │
    ▼
수요일 Quick Screen (30분)
    │
    ├─ Score 3.0 미만 → SKIP
    ├─ Score 3.0~3.4 → 시장 관찰 / 증빙 보강
    └─ Score 3.5+ → 즉시 지원 / 네트워킹 먼저
                │
                ▼
        금요일 지원 스프린트 (1시간)
                │
                ├─ Deep Dive
                ├─ PDF 생성
                ├─ Priority Lane (4.0+)
                └─ 직접 제출
                        │
                        ▼
                월간 Calibration (1시간)
                        │
                        └─ Gap 분석 → 필터 조정 → 다음 달 반복
```

---

## 핵심 원칙

1. **양보다 질.** 하루에 1개 좋은 지원이 10개 스팸 지원보다 낫습니다.
2. **Score 3.0 미만은 바로 스킵.** 시간 낭비하지 마세요.
3. **매주 금요일이 D-day.** 분석은 평일에, 지원은 금요일에 몰아서.
4. **월 1회는 반드시 돌아보기.** 데이터 없이 감으로 하면 시스템이 아닙니다.
5. **career-ops는 절대 자동 제출하지 않습니다.** 최종 버튼은 항상 사용자가 누릅니다.
